# Product Description

This document says what the product is and does, in the personas' terms: the core
capabilities, how they compose, and the boundaries of the product (what it
deliberately is not). It is the parent context for every FRD — features describe
behavior; this describes the whole.

## Known shape (placeholder — to confirm)

> Vinxi is a **reality operating system** platform composed of: a **kernel**, a
> **data fabric**, an **ontology**, an **agentic plane**, and a **developer
> experience** layer.

This is the working shape carried from early design conversations. It is **not
approved product intent**: the product owner must confirm, correct, and expand it
(what each part is, for whom, and where the boundaries lie) before any requirement
cites it.
