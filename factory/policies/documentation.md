# Documentation Policy

The three documentation classes and the front-matter contract are defined in
`docs/README.md`; this policy makes them enforceable. Exception process for every
rule (unless marked "None"): ADR + owner approval.

## 1. Change authority follows the doc class

**Rule.** Descriptive docs: agents update directly. Architectural docs: anyone
proposes; review required before merge. Product-intent docs: agents propose; a
human approves. Class-to-directory mapping: `docs/README.md`.

- **Applies to.** `docs/**`
- **Enforcement.** CODEOWNERS review routing per directory; `docs/product/**`
  requires human approval before merge.
- **Exceptions.** None.
- **Owner.** factory-operators (product-intent approvals: product owner)

## 2. Front matter is mandatory

**Rule.** Every file under `docs/` carries the six-key YAML front matter (`title`,
`summary`, `owners`, `applies_to`, `status`, `last_verified`) defined in
`docs/README.md`. A doc without front matter is undiscoverable and fails CI.

- **Applies to.** `docs/**/*.md`
- **Enforcement.** Review until the lint lands (planned: front-matter lint in the
  CI quality workflow, plus a strict mode in `tools/agent/generate-indexes` — the
  tool currently tolerates missing or malformed keys silently).
- **Exceptions.** None.
- **Owner.** factory-operators

## 3. Self-healing docs: code and docs change together

**Rule.** A code change that alters documented behavior updates the corresponding
doc in the same WO. Find affected docs by matching changed paths against docs'
`applies_to` globs. A stale doc discovered without capacity to fix becomes a
follow-up WO or feedback record — never silence.

- **Applies to.** `docs/**` + all product code
- **Enforcement.** Convergence gate compares landed reality against records
  (`.factory/README.md` §4); final review checks `applies_to` matches for the diff.
  Mechanical `applies_to` matcher planned in `tools/agent/`.
- **Exceptions.** ADR + owner approval.
- **Owner.** factory-operators

## 4. Generated files are never hand-edited

**Rule.** Every `*.generated.md` begins with the line
`<!-- This file is generated by tools/agent/generate-indexes. Do not edit. -->`
and is written only by `tools/agent/generate-indexes`. Source of truth is the
inputs (test files, work orders, blueprints), never the index.

- **Applies to.** `.factory/indexes/**`, `**/*.generated.md`
- **Enforcement.** CI regenerates and diffs; a dirty diff fails. These paths are
  low tier (`.factory/config.yaml`) because the tool, not judgment, produces them.
- **Exceptions.** None.
- **Owner.** factory-operators

## 5. Rejected approaches are recorded

**Rule.** Approaches tried and abandoned during a WO are recorded in the WO's
feedback record (`rejected_approaches` in `.factory/feedback/<YYYY-MM>/WO-*.yaml`,
template: `.factory/templates/feedback.yaml`) with the reason, so future agents and
architecture sweeps do not re-propose them. Rejected product-level ideas route to
`docs/product/` as proposals, never absorbed silently (`.factory/README.md` §8).

- **Applies to.** `.factory/feedback/**`
- **Enforcement.** finish-work-order checklist item; factory-learn synthesis reads
  these records.
- **Exceptions.** None — a WO with nothing rejected writes nothing.
- **Owner.** factory-operators
